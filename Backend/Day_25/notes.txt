Backend Day_25, Moodify Day-1 : Moodify project initiated, created basic folder structure and insatalling required pacakges, and set up basic mood detection functionality using google's mediapipe library.

Backend Day_25, Moodify Day-2 : Implemented authentication and user management features, allowing users to create accounts and log in securely. Integrated JWT for token-based authentication.

Backend Day_25, Moodify Day-3 : Introduced middlewares to the application for handling authentication and authorization, ensuring that only authenticated users can access certain routes and implemented token blacklisting using Mongo DB for enhanced security, allowing users to log out and invalidate their tokens. 

Backend Day_25, Moodify Day-4 : Introduced Redis for token blacklisting, allowing for efficient storage and retrieval of blacklisted tokens. Implemented token expiration handling to ensure that expired tokens are automatically removed from the blacklist.

Backend Day_25, Moodify Day-5 : Integrated the frontend with the backend to support user authentication (register, login, logout). Implemented protected routes to control access based on authentication state and followed a 4-layer architecture in React for better organization and separation of concerns.

Backend Day_25, Moodify Day-6 : Implemented song upload functionality, allowing users to upload songs along with their associated moods with the help of Multer middleware for handling multipart/form-data and stored all the data on imagekit. Created a song context to manage the state of the currently playing song and its associated information, such as the song URL, poster URL, title, and mood. This context allows for easy access and updates to the song information across different components in the application.

Backend Day_25, Moodify Additional Features : Added shuffle and repeat functionality to the music player, added songcard component to display song information of currently playing song, added upload song form to allow users to upload songs, and added a playlist component to display the list of uploaded songs. Also added a logout button to allow users to log out of their accounts and implemented the functionality to clear the song context and blacklist the user's token upon logout.
